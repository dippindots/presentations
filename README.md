# presentations

Static presentation websites built from Marp and published on GitHub Pages.

Each presentation is published at:

```text
/YYYY-MM-DD/short-name/
```

For a project Pages site, the full URL looks like:

```text
https://<your-user>.github.io/presentations/YYYY-MM-DD/short-name/
```

## Repository layout

```text
slides/
  _template/
  2026-05-11-my-talk/
    deck.md
    assets/
themes/
tools/
public/               # generated by the build
.github/workflows/
```

## Getting started

Install dependencies:

```bash
npm ci
```

Create a new deck scaffold:

```bash
npm run new -- 2026-05-11 my-talk
```

Build the site:

```bash
npm run build
```

Open the generated deck offline:

```bash
open public/2026-05-11/my-talk/index.html
```

Or preview the generated site locally:

```bash
python3 -m http.server 8000 -d public
```

## Authoring rules

- Store each deck in `slides/YYYY-MM-DD-short-name/`.
- Keep deck assets local in that deck's `assets/` directory.
- Use only local images, fonts, CSS, and scripts if you want reliable offline access.
- Keep the deck's published URL stable once shared.

## Deployment

GitHub Actions builds the site and deploys `public/` to GitHub Pages on pushes to `main`.
